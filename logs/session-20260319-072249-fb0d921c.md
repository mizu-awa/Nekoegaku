## 原因

スキャンアルゴリズムが「黒線の最初の連続ブロック」を求めているのが問題です。

### 直線部分（正常に動く場所）

```
上から走査 → ████ ← topStart（外縁）
           ████
           ████ ← topEnd（内縁）
           　　　← break（透明に戻る = 猫の内側）
```
→ 中央 = (topStart + topEnd) / 2 ≒ 線の中心 ✅

### 問題が起きる接続部

**鼻の先端（顔の左端）・しっぽの頂点**では、線が折り返してループしているため、スキャン方向において「外縁の線」と「内縁の線」が**同一の連続ブロック**に見える：

```
上から走査 → ████  ← topStart（鼻の上端）
           ████
           ████  ← この辺りは曲線の折り返し部分で、透明な隙間がない
           ████
           ████  ← topEnd（鼻の曲線の折り返し底まで全部ひとつながり）
           　　　← break
```

→ 中央 = 鼻全体の高さの半分 = 鼻の先端から大きく外れた位置になる ❌

### おしりと足の間

おしりの線が急角度で上昇する箇所も同様に、下からスキャンした連続ブロックが「足の線＋おしりの線」をまたぐ場合があり、bottomFound が大きくズレる。

---

### まとめ

「黒線の中央」アルゴリズムは **細い直線ストロークには有効**ですが、**線が鋭くカーブして折り返す箇所（先端・急カーブ）** では、1回のスキャンで拾うブロックが線の太さをはるかに超えた高さになり、中央位置が大きくズレます。これが3か所すべての不自然さの共通の原因です。
