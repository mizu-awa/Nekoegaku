## 1. 猫の形状（パス/輪郭）定義

### 基本構造
猫は **2本の輪郭線** で構成されています：

| 線 | 範囲 | 関数 | 役割 |
|---|---|---|---|
| **上の線（頭〜背中〜しっぽ）** | noseX(0.025) → tailX(0.96) | `getTopY(xRatio, params, W, H)` | 頭、耳、背中、しっぽの上縁 |
| **下の線（あご〜足〜しっぽ付け根）** | noseX(0.025) → tailX(0.96) | `getBottomY(xRatio, params, W, H)` | あご、足、しっぽ付け根の下縁 |

### 上の線の計算式
**ファイル**: `/c/dev/nekoegaku/js/game.js` 行78-101

```
getTopY(xRatio, params, W, H) {
  1. LUTベースライン（元画像から抽出）を優先使用
  2. フォールバック: 数式ベースライン
     
  数式: y = baseline - ears - tail
  
  baseline = backY + faceY
  - backY: 背中のベースY (0.25 * H)
  - faceY: 顔の張り出し量 = faceT * (noseY - backY)
           faceT = max(0, 1 - t*faceDecayRate)
           * t は正規化された位置 (0〜1)
           * faceDecayRate = 12（設定値）
           * 左端のみ下方向へ張り出し
  
  ears: ガウシアン関数（2つ）
  tail: スムーズステップ関数
}
```

**設定値** (`config/settings.json`):
- `noseX`: 0.025 (左端)
- `tailX`: 0.96 (右端)
- `noseY`: 0.49
- `backY`: 0.25
- `earCenterX`: 0.175

### 下の線の計算式
**ファイル**: `/c/dev/nekoegaku/js/game.js` 行106-147

```
getBottomY(xRatio, params, W, H) {
  1. LUTベースライン（元画像から抽出）を優先使用
  2. フォールバック: 数式ベースライン
  
  数式: y = baseline + feet
  
  baseline の構成（区間ごと）:
  - 顔領域 (t < faceEndT): 
    noseY からスムーズに bottomY へ遷移
    faceEndT = 0.10
  
  - 中央領域 (faceEndT ≤ t ≤ buttRiseStart):
    bottomY で水平 (0.65 * H)
  
  - お尻領域 (buttRiseStart < t ≤ buttX):
    bottomY から backY へスムーズ上昇
    buttRiseStart = 0.90, buttX = 0.89
    スムーズステップ: u = u² * (3 - 2u)
  
  - しっぽ領域 (t > buttX):
    backY からしっぽ先端へ向かって上昇
  
  feet: sin波にエンベロープを乗じたもの
}
```

**設定値**:
- `bottomY`: 0.65
- `faceEndT`: 0.10
- `buttRiseStart`: 0.90
- `buttX`: 0.89
- `feetStartT`: 0.12
- `feetEndT`: 0.88
- `feetFadeWidth`: 0.04

## 3. スコア計算

### 計算フロー

**ファイル**: `/c/dev/nekoegaku/js/game.js` 行560-591 (`calculateScore`関数)

```javascript
function calculateScore() {
  // 1. 画像から抽出した正解の輪郭 (refTopY, refBottomY) と
  //    プレイヤーが調整したパラメータから計算した輪郭を比較
  
  // 2. サンプリング: SC.sampleStep = 2 (2pxごと)
  const startPx = Math.floor(CAT.noseX * W);  // = 20px
  const endPx = Math.ceil(CAT.tailX * W);     // = 768px
  
  for (let px = startPx; px <= endPx; px += SC.sampleStep) {
    const xr = px / W;
    
    // プレイヤーの輪郭
    const pTop = getTopY(xr, player, W, H);
    const pBot = getBottomY(xr, player, W, H);
    
    // 正解の輪郭
    const tTop = refTopY[idx];
    const tBot = refBottomY[idx];
    idx++;
    
    // 二乗誤差を累積
    totalError += (tTop - pTop) ** 2;
    totalError += (tBot - pBot) ** 2;
    count += 2;  // 上下2点をカウント
  }
  
  // 3. RMSE（平均二乗誤差の平方根）を計算
  const rmse = Math.sqrt(totalError / count);
  
  // 4. スコア算出
  const score = Math.max(0, Math.round(
    SC.maxScore * Math.max(0, 1 - rmse / SC.maxRmse)
  ));
  // maxScore = 100, maxRmse = 80
  // RMSE = 0 → score = 100
  // RMSE ≥ 80 → score = 0
  // 線形逓減
  
  return score;
}
```

### スコア計算の基準（何を測るか）

1. **計測対象**: 上線と下線の **Y座標差分** 
2. **計測点**: noseX(0.025) → tailX(0.96) 間を **2pxごと** にサンプリング
3. **正解データソース**:
   - `refTopY`, `refBottomY` ← `extractContours()` で画像から抽出
   - 元画像（head.png, tail.png）のアルファチャンネルを走査
4. **距離関数**: ユークリッド距離（二乗誤差）
5. **正規化方法**: RMSE（ピクセル単位）

**設定値**:
```json
"scoring": {
  "maxRmse": 80,           // RMSE = 80 で 0 点
  "maxScore": 100,         // 最高点
  "smoothingRadius": 3,    // 抽出後の輪郭をスムージング
  "alphaThreshold": 128,   // アルファ値がこれ以上で非透明と判定
  "sampleStep": 2          // 2pxごとにサンプリング

## 実装上の工夫

### 1. LUTベースライン（Baseline Look-Up Table）
- 手描きイラスト（head.png, tail.png）から一度だけ輪郭を抽出
- 抽出した上下の線を「ベースライン」として保存
- 実行時は LUT から線形補間で Y座標を取得（計算量削減）
- フォールバック: LUT がなければ数式で計算

**実装**:
```javascript
// getTopY/getBottomY では LUT優先
const baseline = interpolateBaseline(refTopBaseline, xRatio);
if (baseline !== null) {
  const ears = getEarContribution(xRatio, params, W);
  const tail = getTailContribution(xRatio, params, W, H);
  return baseline - ears - tail;  // LUT版
}
// フォールバック: 数式ベースライン
```

### 2. スムーズステップ関数
- しっぽやお尻の形状遷移に使用
- `u² * (3 - 2u)` で滑らかな曲線を実現

### 3. ガウシアン関数（耳）
```
ear = height * exp(-((x - center)² / (2 * sigma²)))
```
- 形状: 高さ、位置（gap）、鋭さ（width）をパラメータ化
- 2つの耳を独立に計算して加算

### 4. Sin波エンベロープ（足）
```
feet = amplitude * sin(frequency * 2π * t + phase) × envelope
envelope = feetEnvLeft × feetEnvRight
```
- エンベロープ: 顔・お尻領域で フェードアウト（不自然な波を防止）
