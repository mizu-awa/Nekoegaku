## `getTopYNoEars()` の仕様

[game.js:411-413](js/game.js#L411-L413)

```js
function getTopYNoEars(xRatio, params, W, H) {
  return getTopY(xRatio, params, W, H) + getEarContribution(xRatio, params, W);
}
```

非常にシンプルで、**2つの関数の合成**です。

---

### やっていること

`getTopY()` は上の線全体のY座標を返しますが、その中で**耳の凹み（上方向への突出）を引き算**しています（[game.js:89](js/game.js#L89)）:

```js
return baseline - ears - tail;  // ears分だけ上に凹む
```

`getTopYNoEars()` は、その結果に `getEarContribution()` を**足し戻す**ことで、耳の凹みを**キャンセル**しています:

```
getTopY (= baseline - ears - tail)  +  ears  =  baseline - tail
```

つまり「**耳がない状態の上の線**」を返します。

---

### `getEarContribution()` の中身

[game.js:391-398](js/game.js#L391-L398)

2つの耳それぞれに**ガウス関数（正規分布の釣鐘型）** を使っています:

```js
params.earHeight * Math.exp(-((x - earCenterX)²) / (2 * earWidth²))
```

- 耳の中心からの距離に応じて滑らかに減衰
- `earHeight` が耳の高さ、`earWidth` が耳の広がり幅
- 2つの耳の寄与を単純に足し合わせ

---

### 線が納得いかない形になる原因として考えられること

1. **耳の裾が広すぎる** — `earWidth` が大きいと、ガウスの裾が鼻付近まで影響し、「耳を除いた」つもりでも完全にキャンセルしきれていないように見える（実際は数学的には正確にキャンセルしている）

2. **ベースライン自体の問題** — `getTopY()` のLUTベースラインまたはフォールバック数式（`faceDecayRate` による減衰カーブ）が顔の形に合っていない。LUTがある場合は元画像依存、ない場合は [game.js:95-96](js/game.js#L95-L96) の指数的減衰で顔の膨らみを作っているので、そのカーブ自体が問題かもしれません

3. **tail成分が残っている** — `getTopYNoEars` は耳だけ除去して `tail` 成分は残ったままです。顔の領域でも `getTailContribution()` がわずかに効いている可能性があります

具体的にどのあたりの形が気になりますか？ 鼻付近のカーブ、額の丸み、耳との接続部分など、場所がわかればピンポイントで原因を特定できます。
