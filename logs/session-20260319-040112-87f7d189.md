## 問題のある部分（ユーザーの指摘）

コード分析から、以下の問題が確認できました:

### 1. 尻尾の問題（閉じた輪が形成される）
**原因**: `getBottomY()` の計算

現在の実装:
- `tailTipY = backY - tailHeight` （尻尾の先端はY方向に高くなる）
- 下の線は`baseY + env × (...)`で計算
- `env = sin(π × t)`が右端(t=1)で0になる
- **結果**: 上の線と下の線が両方とも右端で同じ高さに収束し、尻尾が閉じた輪になる

### 2. 足の問題（右上に向かって傾く）
**原因**: `getBottomY()` の計算

下の線全体が左から右にかけて上昇（baseYが線形補間で上がる）:
- `baseY = noseY × (1 - t) + tailTipY × t`
- 尻尾が高くなるほど、下の線全体が上昇
- **結果**: 足（sin波で作られた）が右上に向かって傾く

### 3. 耳の問題（左下に傾く）
**原因**: `getTopY()` の顔の輪郭計算

- `faceT = max(0, 1 - t*6)`は左端(t≈0)でのみ1で、右にいくに従い0になる
- この顔の張り出しが減衰すると、耳も相対的に下がったように見える
- または、耳の標準偏差の計算時に座標系の非対称性

実装コード:
- `ear1X = (0.17 - earGap/W) × W` （earGap/Wはピクセル単位をW基準に変換）
- `ear2X = (0.17 + earGap/W) × W`
- 耳の計算自体は対称的だが、全体の背景構造（顔の輪郭など）の影響

  if (t > 1) return CAT.bottomY * H;  // お尻より右は足の底ラインで固定

    const ft = t / faceEndT;  // 0→1
    const smooth = ft * ft * (3 - 2 * ft);  // smoothstep
