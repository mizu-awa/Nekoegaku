## 見つかった問題点

### 1. 最新ファイルの検出が全プロジェクト横断 (最も危険)

[save_session.py:52-55](save_session.py#L52-L55)

```python
def find_latest_jsonl(projects_dir: Path) -> Path | None:
    files = find_all_jsonl(projects_dir)
    return files[0] if files else None
```

`~/.claude/projects/` 以下の**全プロジェクト**を `st_mtime` で並べて最新を返します。別プロジェクトでセッションがあった直後にこのスクリプトを実行すると、**意図した会話ではなく別プロジェクトのログが保存されます**。引数なし実行時に起こりやすいです。

### 2. アクティブセッション中の実行でメッセージが欠落

スクリプト実行時点でまだClaudeがJSONLに書き込み中の場合、最後のメッセージが取り込まれない可能性があります。

### 3. `isSidechain` エントリの除外 ([save_session.py:202-203](save_session.py#L202-L203))

```python
if entry.get("isSidechain"):
    continue
```

サブエージェント（Agent tool）による会話は丸ごとスキップされます。「ログが減った」と感じる原因になります。

### 4. `lines.insert(10, ...)` のハードコード ([save_session.py:230](save_session.py#L230))

ヘッダー構造が変わると挿入位置がずれてテーブルが壊れます。

引数なし実行時に現在の作業ディレクトリからプロジェクトを絞り込む修正が効果的です：

```python
def find_latest_jsonl(projects_dir: Path, cwd: Path | None = None) -> Path | None:
    files = find_all_jsonl(projects_dir)
    if cwd and files:
        # 現在のcwdに対応するプロジェクトフォルダを優先

        matched = [f for f in files if cwd_encoded in f.parent.name.lower()]
        if matched:
            return matched[0]
    return files[0] if files else None
```
