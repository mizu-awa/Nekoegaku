## ねこえがく — ゲーム設計案

### コンセプト
ねこの体の各パーツが**数学関数**で表現されており、プレイヤーがパラメータのつまみを調整して「正解のねこ」に近づける。

### ねこの構成要素と対応する関数

画像を見ると、ねこは以下のパーツに分解できます：

| パーツ | 関数のイメージ | 調整パラメータ |
|--------|---------------|---------------|
| **耳** | 2つのピークを持つ関数（例: `A₁·exp(-(x-μ₁)²/σ₁²) + A₂·exp(-(x-μ₂)²/σ₂²)`） | 高さA、位置μ、幅σ × 2つ |
| **背中** | 直線～緩やかなカーブ（`a·x + b` or 定数） | 傾きa、高さb |
| **足（下側の波）** | sin波の合成（`A·sin(ω·x + φ) + offset`） | 振幅A、周波数ω、位相φ、オフセット |
| **顔の輪郭** | 斜めの直線2本（あご→耳の下） | 角度、長さ |
| **しっぽ** | 上に跳ねるカーブ（`A·(x-x₀)²` や `A·exp(k·x)` など） | 高さA、カーブの急さk |

### 議論したいポイント

**1. パーツの分割粒度**
- **案A**: 全部一筆書き（1つの関数で全体を表現）→ 超難しいが関数1つ
- **案B**: 上記のようにパーツごとに別関数 → 直感的でわかりやすい
- **案C**: 「上の輪郭線」と「下の輪郭線」の2本だけ → シンプル

画像を見ると**案C（上の線・下の線）**が自然に見えます。上の線 = 耳→背中→しっぽ、下の線 = 顔の下→足の波。

**2. 難易度の幅**
- 足の本数が多い = 周波数が高い → 難しいステージほど多足ねこ？
- パラメータの数を増やす（sin波だけ → sin + cos合成 → フーリエ級数的に）

**3. スコア計算**
- 正解の曲線とプレイヤーの曲線の**面積差**（数値積分）
- またはサンプル点での**平均二乗誤差**

**4. 技術スタック**
- ブラウザゲーム（HTML + Canvas/SVG + JavaScript）が手軽
- スライダーUIはHTML rangeで十分
